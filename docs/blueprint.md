## Service One-Liner

**MarryTone**은 퍼스널 컬러·골격·취향을 결합해 예비부부의 스드메 선택을 "추천 + 비교 + 기록"으로 끝까지 돕는 웨딩 스타일 코치다.

---

## 해결하는 문제

1. 선택지 과다 → 의사결정 피로 누적
2. 내게 맞는 기준 없이 SNS·커뮤니티 정보에 흔들림
3. 예비신랑·예비신부의 취향 조율 이력이 남지 않음

---

## 사용자 유형

| 유형 | 상황 | 주요 니즈 | 핵심 UX 요구사항 |
|---|---|---|---|
| 예비신부 (주도형) | 드레스·메이크업 중심 탐색 | 체형·톤 기반 후보 압축 | 모바일 한손 조작, 즉시 저장 |
| 예비신랑 (협업형) | 턱시도·촬영 컨셉 참여 | 파트너와 비교·합의 | 커플 공유 보드, 코멘트 |
| 예비부부 (시간부족형) | 직장 병행 준비 | 짧은 세션 내 결정 | 5분 요약 추천, 체크리스트 |

## 핵심 기능 (MVP)

| 기능 | 설명 |
|---|---|
| 진단 게이트 | 기존 진단 경험 유무 확인 → 경험자/비경험자 경로 분기 |
| 경험자 입력 | 퍼스널 컬러·골격 결과 직접 입력 또는 결과지 업로드 |
| 비경험자 진단 | 맨얼굴 사진(퍼스널 컬러) + 전신 사진(골격) 업로드 → 자동 진단 |
| 저신뢰 C정책 | confidence 미달 → 재촬영 유도 → 이후에도 미달 → 설문 기반 수동 전환 |
| AI 스타일 추천 | 카테고리별 최대 3안 제안 + 메인 1안 + 추천 이유 설명 |
| 스타일 보관함 | 카드 저장, 태그, 메모, 상태값(검토중/보류/확정) |
| 비교 보드 | 최대 4개 후보 병렬 비교, 장단점 자동 요약 |
| 커플 협업 | 공유 링크, 코멘트, 합의 보드 수동 확정 |
| 준비 트래커 | 카테고리별 TODO·데드라인, 완료율 100% → MVP 완료 |

## 추천 카테고리

드레스 / 턱시도 / 헤어 / 메이크업 / 촬영 컨셉 / 예식장 환경

각 카테고리: **최대 3안 제안 + 메인 1안 강조**

---

## AI 추천·진단 흐름

```
사전 질문 게이트
    ├── 경험자 → 결과 입력/업로드 → 유효성 검증
    └── 비경험자 → 사진 업로드 → 자동 진단
              └── 저신뢰(C정책) → 재촬영 유도
                        └── 재시도 후에도 미달 → 설문 기반 수동 진단

통합 프로필 생성 (진단결과 + 분위기/무드/촬영 취향)
    ↓
하드 필터링 + 후보 스코어링 (규칙 점수 + 선호 점수)
    ↓
카테고리별 최대 3안 + 메인 1안
    ↓
LLM 설명 생성 (추천 이유, 주의 포인트, 대안 가이드)
    ↓
실패 폴백: 규칙 기반 상위 3안 + "추천 근거 생성 중" 표시 + 1회 재시도
```

## 저장·비교·기록 설계

- **저장 단위**: 스타일 카드 (`category`, `fit_score`, `mood_tags`, `memo`, `status`)
- **비교 단위**: 동일 카테고리 2~4개 후보 + 자동 비교 포인트 (핏/무드/실용성/예산)
- **기록 단위**: 의사결정 이벤트 로그 (누가, 언제, 어떤 이유로 상태 변경)
- **되돌리기**: 최근 20개 상태 변경 undo 가능
- **완료 단위**: 체크리스트 진행률 100% → `MVP_COMPLETE` 이벤트

## ML Sidecar API 계약 (내부)

| Endpoint | Input | Output |
|---|---|---|
| `POST /diagnosis/personal-color` | 맨얼굴 이미지 | 피부·홍채 대표색 + PCCS/KS 코드 + confidence |
| `POST /diagnosis/body-measurements` | 전신 이미지 + 키 | 신체 측정치 |
| `POST /diagnosis/skeleton-type` | 측정치·특징 | straight/wave/natural + confidence + 근거 |

### 구현 스택

- `FastAPI` (MIT)
- `MediaPipe` (Apache-2.0) — 얼굴 랜드마크·홍채
- `OpenCV >= 4.5` (Apache-2.0) — 이미지 처리·색공간 변환
- `numpy + scikit-learn KMeans` (BSD-3-Clause) — 대표색 추출
